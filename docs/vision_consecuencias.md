## 🏗️ ¿Qué estamos construyendo realmente?

No estamos desarrollando otra aplicación sobre la Web2, ni un protocolo Web3 inflado con tokens especulativos. Estamos pariendo **IAON (Iberá Autonomous Overlay Network)**: un nuevo estándar y sistema operativo para la web descentralizada que corre directamente sobre el metal (hardware real), desacoplado por completo de la infraestructura y el direccionamiento corporativo.

### Arquitectura en Capas

+-------------------------------------------------------+
| CRDT State Sync & Pn-Counters | -> Capa de Datos
+-------------------------------------------------------+
| DIDs secp256k1 + Adaptive 16-32bit PoW | -> Capa de Identidad
+-------------------------------------------------------+
| Kademlia DHT (XOR Metric Tables over UDP) | -> Capa de Ruteo
+-------------------------------------------------------+
| Noise Framework (IK_25519_ChaChaPoly_BLAKE2b) | -> Capa de Encriptación
+-------------------------------------------------------+
| Physical Metal (TV Boxes, Recycled Xeons) | -> Capa de Hardware
+-------------------------------------------------------+

## 👥 Consecuencias para el Usuario Común

### ⚡ Fortalezas Reales (Por qué importa en la trinchera)

| Fortaleza | Descripción |
|-----------|-------------|
| **Resiliencia Asimétrica** | Si perdés tu teléfono, tu identidad criptográfica se recupera desde nodos vecinos en segundos. |
| **Privacidad de Metadatos** | ISP no puede inspeccionar tráfico ni mapear relaciones del enjambre. |
| **Independencia del Apagón Corporativo** | La red superpuesta sigue operando aunque AWS, Google o Cloudflare colapsen. |
| **Democratización del Hardware Obsoleto** | Un Poco F1 con pantalla rota tiene el mismo peso que un servidor Xeon. |
| **Propiedad Atómica de los Datos** | Los datos mutan localmente y convergen matemáticamente en un join-semilattice (S_A ⊔ S_B). |

### ⚠️ Costos Reales y Peajes Técnicos

| Cosa | Impacto |
|------|---------|
| **Consumo de Recursos** | Mantener k-buckets activos bajo alta intermitencia (churn) incrementa batería y payloads UDP. |
| **Barrera de UX** | El usuario debe entender que es un nodo activo, no un consumidor pasivo. |
| **Problema de Cold-Start** | El arranque inicial requiere nodos semillas estables (Xeon del búnker). |
